Lathe.models.LeastSquare — Method

Least Squares is ideal for predicting continous features. Many models use Least Squares as a base to build off of.

x = [7,6,5,6,5]

y = [3.4.5.6.3]

xtrain = [7,5,4,5,3,5,7,8]

Type = :LIN

model = Lathe.models.LeastSquare(x,y,Type)

y_pred = Lathe.models.predict(model,xtrain)

HYPER PARAMETERS

Type <- Type determines which Linear Least Square algorithm to use, :LIN, :OLS, :WLS, and :GLS are the three options.

  • :LIN = Linear Least Square Regression

  • :OLS = Ordinary Least Squares

  • :WLS = Weighted Least Squares

  • :GLS = General Least Squares

==Functions==

predict(xt) <- Returns a prediction from the model based on the xtrain value passed (xt)

Lathe.models.Pipeline — Method

Pipelines can contain a predictable Lathe model with preprocessing that occurs automatically. This is done by putting X array processing methods into the iterable steps, and then putting your Lathe model in.

==PARAMETERS==

[steps] <- An iterable list of methods to call for X modification. These mutations should have ALREADY BEEN MADE TO THE TRAIN X.

pipl = Pipeline([StandardScalar(),LinearRegression(trainX,trainy)])

Lathe.models.PowerLog — Method

A powerlog can be used to perform a one-tailed test, as well as get the proper sample size for a testing population.

==PARAMETERS==

p1 <- A Float64 percentage representing the probability of scenario one.

p2 <- A Float64 percentage representing the probability of scenario two. These two probability values should follow these guidelines: p1 = p1 + x = p2

alpha = 0.05 <- Sets an alpha value

Returns power, sample_size
